Glossary
========

.. glossary::

   ADAS
      Advanced Driver Assistance Systems. Electronic systems that assist drivers in driving and parking functions.

   AEB
      Automatic Emergency Braking. System that automatically applies brakes to prevent or mitigate collisions.

   ASIL
      Automotive Safety Integrity Level. ISO 26262 risk classification from QM (non-safety) to D (highest risk).

   ASPICE
      Automotive SPICE (Software Process Improvement and Capability Determination). Process assessment model for automotive software.

   AUTOSAR
      Automotive Open System Architecture. Standardized automotive software architecture.

   CAN
      Controller Area Network. Robust vehicle bus standard for microcontrollers and devices to communicate.

   ECU
      Electronic Control Unit. Embedded system that controls one or more electrical systems in a vehicle.

   FMEA
      Failure Mode and Effects Analysis. Systematic method for identifying potential failures and their effects.

   FTA
      Fault Tree Analysis. Top-down, deductive failure analysis using Boolean logic.

   HARA
      Hazard Analysis and Risk Assessment. ISO 26262 process for identifying hazards and assessing risks.

   HIL
      Hardware-in-the-Loop. Testing method where real hardware is connected to simulated environment.

   ISO 26262
      International standard for functional safety of electrical/electronic systems in road vehicles.

   LLD
      Low-Level Design. Detailed component specifications and implementation details.

   MC/DC
      Modified Condition/Decision Coverage. Code coverage metric requiring testing of all conditions independently.

   MISRA C
      Motor Industry Software Reliability Association C. Coding standard for safety-critical C programming.

   OBD
      On-Board Diagnostics. Vehicle self-diagnostic and reporting capability.

   QM
      Quality Management. Non-safety-related classification in ISO 26262.

   SIL
      Software-in-the-Loop. Testing method where software runs on development PC with virtual environment.

   TTC
      Time-to-Collision. Metric for collision threat assessment.

   V-Model
      Software development and testing methodology emphasizing verification and validation at each stage.

   VIL
      Vehicle-in-the-Loop. Testing with complete vehicle on dynamometer in controlled environment.
